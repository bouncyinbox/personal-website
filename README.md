# Personal website

A minimal, text-first personal site for a Senior Engineering Manager. Built with [Astro](https://astro.build) and plain CSS. Static output, deployed to Vercel.

## Quickstart

```sh
pnpm install
pnpm dev      # local dev server on http://localhost:4321
pnpm build    # static site in ./dist
pnpm preview  # serve the built site locally
```

Node 20+ is required (see `.nvmrc`).

## Filling in placeholders

Every personal detail is wrapped in `[PLACEHOLDER: ...]`. The single-pass list lives in [CONTENT.md](./CONTENT.md). To audit everything that still needs a value:

```sh
grep -rn "PLACEHOLDER" src public scripts astro.config.mjs README.md CONTENT.md
```

## Writing a new blog post

1. Create a file in `src/content/blog/`, e.g. `src/content/blog/my-post.md`.
2. Frontmatter:

   ```yaml
   ---
   title: "My post"
   description: "One-line summary used in meta tags and the index."
   pubDate: 2026-02-04
   updatedDate: 2026-02-10   # optional
   draft: false              # set true to hide from the index and routes
   ---
   ```

3. Write Markdown. Headings (`##`, `###`), lists, code fences, blockquotes вЂ” all styled.

The blog index sorts non-draft posts by `pubDate` descending. Reading time is auto-computed (в‰€220 words/minute).

## Comments (later)

There's an integration point in `src/pages/blog/[...slug].astro` вЂ” a commented-out `<Comments />` slot at the end of the article. Drop in [Giscus](https://giscus.app) (or similar) when you want comments. No backend required, no rebuild plumbing changes.

## Printing the CV

1. Visit `/cv`.
2. Click **Print / Save as PDF** (or use the browser print dialog).
3. Choose **Save as PDF**, paper size **Letter** or **A4**, default margins.

The print stylesheet hides the site nav, footer, and the print button itself; everything else is tuned to fit a representative content set on a single page at default browser scaling.

## Plausible Analytics

Off by default. To turn it on:

1. Edit `src/config.ts` and set `PLAUSIBLE_ENABLED = true`.
2. Confirm `domain` is the production hostname вЂ” it's reused as `data-domain` on the script tag.
3. Rebuild and deploy.

## Deploying to Vercel

1. Push this repo to GitHub.
2. In Vercel, **Add New в†’ Project в†’ Import Git Repository**, pick the repo.
3. Framework preset auto-detects as **Astro**. Build command `pnpm build`, output `dist`.
4. Add your custom domain under **Settings в†’ Domains**.
5. Update `site` in `astro.config.mjs` and `domain` in `src/config.ts` to match the real domain so canonical URLs and the sitemap resolve correctly.

The site is fully static вЂ” no server functions, no environment variables required.

## Typography

The site uses a system font stack вЂ” `ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto, вЂ¦` for body text and `ui-serif, Georgia, вЂ¦` for headings.

**Why no webfont?** A system stack ships zero bytes, has zero layout shift, and renders crisply on every platform. The serif headings give a touch of warmth without committing to a typographic identity that would need to be designed and maintained. If you want a webfont later, drop a single self-hosted `woff2` into `public/fonts/`, add `@font-face` and `font-display: swap` to `src/styles/global.css`, and swap in the family вЂ” keep it to one face.
